# Website Editor Admin Page — Functional Specification

เอกสารนี้อธิบายเฉพาะ “ระบบ/ฟังก์ชัน” ของหน้าแก้ไขหน้าเว็บ SHOW OFF สำหรับส่งให้นักออกแบบ UI ไปออกแบบหน้าตาเอง

เอกสารนี้ไม่กำหนดสี เลย์เอาต์ สไตล์ ปุ่ม การ์ด หรือรูปแบบดีไซน์ใด ๆ

---

## 1. เป้าหมายของหน้านี้

หน้า Website Editor คือหน้าหลังบ้านสำหรับให้แอดมินแก้ไขข้อมูลที่แสดงบนเว็บหน้าบ้าน โดยเน้นเฉพาะส่วนที่ต้องเปลี่ยนบ่อยและเกี่ยวข้องกับการขายจริง

สิ่งที่หน้านี้ต้องครอบคลุม:

- Cover หน้าเว็บ
- Intro ตอนเข้าเว็บ
- กฎรูปสินค้า
- ข้อมูลสินค้าและหมวดหมู่ที่แสดงบนเว็บ
- การเชื่อมสินค้าเข้ากับระบบสต็อก

หลักสำคัญคือ แอดมินควรแก้ไขข้อมูลจากหลังบ้านได้ โดยไม่ต้องไปแก้โค้ด และไม่ต้องเพิ่มข้อมูลซ้ำหลายหน้า

---

## 2. Home Cover / Cover หน้าเว็บ

ส่วน Cover ใช้สำหรับจัดการรูปภาพหรือวิดีโอที่แสดงบนหน้าเว็บ เช่น หน้า Home หรือส่วน hero/campaign ของเว็บ

ตาม scope ปัจจุบัน Cover ต้องทำได้เฉพาะการจัดการ media เท่านั้น

### สิ่งที่ต้องทำได้

- เพิ่ม cover ใหม่
- แก้ไข cover เดิม
- ลบ cover
- เลือกประเภท media ได้:
  - รูปภาพ
  - วิดีโอ
- อัปโหลดหรือเลือกไฟล์รูปภาพ
- อัปโหลดหรือเลือกไฟล์วิดีโอ
- แสดง preview ก่อนบันทึก
- บันทึกข้อมูล cover

### ข้อมูลที่ควรมีใน cover แต่ละรายการ

- ชื่อรายการ cover สำหรับให้แอดมินรู้ว่าเป็นไฟล์ไหน
- ประเภทไฟล์: image หรือ video
- ไฟล์ media
- สถานะใช้งานหรือไม่ใช้งาน ถ้าระบบต้องการ
- วันที่สร้าง
- วันที่แก้ไขล่าสุด

### ขนาดไฟล์และคำแนะนำที่ต้องแสดงในหลังบ้าน

หลังบ้านควรบอกขนาดไฟล์แนะนำให้ชัดเจน เพื่อให้แอดมินอัปโหลดไฟล์ได้ถูกต้อง

แนวทางที่คุยกัน:

- ใช้ไฟล์ desktop เป็นหลัก
- ยังไม่ต้องแยกไฟล์ mobile ในช่วงแรก
- เมื่อเปิดบนมือถือ ให้เว็บ crop/จัดภาพเองตามระบบหน้าเว็บเดิม
- รองรับรูปภาพและวิดีโอ
- ต้องบอกชนิดไฟล์ที่รองรับ เช่น `.jpg`, `.png`, `.webp`, `.mp4`, `.webm`

### สิ่งที่ไม่ต้องอยู่ใน scope cover ตอนนี้

- ไม่ต้องแก้ headline
- ไม่ต้องแก้ label
- ไม่ต้องแก้ปุ่ม CTA
- ไม่ต้องแก้ link
- ไม่ต้องมีระบบ schedule
- ไม่ต้องมี crop tool ขั้นสูง
- ไม่ต้องมีการตั้งค่า animation

## 3. Intro ตอนเข้าเว็บ

ส่วน Intro ใช้สำหรับจัดการรูปภาพหรือวิดีโอที่แสดงตอนผู้ใช้เข้าเว็บ

ตาม scope ปัจจุบัน Intro ต้องทำได้เฉพาะการจัดการ media เท่านั้น

### สิ่งที่ต้องทำได้

- เพิ่ม intro media
- แก้ไข intro media
- ลบ intro media
- เลือกประเภท media ได้:
  - รูปภาพ
  - วิดีโอ
- อัปโหลดหรือเลือกไฟล์รูปภาพ
- อัปโหลดหรือเลือกไฟล์วิดีโอ
- แสดง preview ก่อนบันทึก
- บันทึกข้อมูล intro

### ข้อมูลที่ควรมีใน intro แต่ละรายการ

- ชื่อ intro สำหรับหลังบ้าน
- ประเภทไฟล์: image หรือ video
- ไฟล์ media
- สถานะใช้งานหรือไม่ใช้งาน ถ้าระบบต้องการ
- วันที่สร้าง
- วันที่แก้ไขล่าสุด

### ขนาดไฟล์และคำแนะนำที่ต้องแสดงในหลังบ้าน

หลังบ้านควรบอกขนาดไฟล์แนะนำให้ชัดเจน:

- รองรับรูปภาพ
- รองรับวิดีโอ
- ระบุชนิดไฟล์ที่รองรับ เช่น `.jpg`, `.png`, `.webp`, `.mp4`, `.webm`
- ระบุขนาดไฟล์หรือสัดส่วนที่แนะนำ

### สิ่งที่ไม่ต้องอยู่ใน scope intro ตอนนี้

- ไม่ต้องตั้งค่าเสียง
- ไม่ต้องตั้งค่า fade
- ไม่ต้องตั้งค่าระยะเวลาเล่น
- ไม่ต้องตั้งค่าแสดงครั้งเดียว
- ไม่ต้องตั้งค่า animation
- ไม่ต้องแก้ logic การเล่น intro จากหน้านี้

หมายเหตุ: logic intro บนเว็บหน้าบ้านให้ใช้ระบบเดิมของเว็บ หน้านี้มีหน้าที่จัดการไฟล์รูปภาพ/วิดีโอเท่านั้น

## 4. Product Image Rule

หน้านี้ต้องบอกกฎของรูปสินค้าให้ชัดเจน เพราะรูปสินค้าของ SHOW OFF มีวิธีใช้งานเฉพาะ

จากที่คุยกัน รูปสินค้าใช้ไฟล์เดียวเป็นหลัก ทั้ง desktop และ mobile

### กฎรูปสินค้าที่ต้องรองรับ

- ใช้รูปสินค้าไฟล์เดียว
- ไม่แยกรูป desktop/mobile
- ขนาดรูปสินค้าแนะนำ: 1920 × 2400 px
- สัดส่วน 4:5
- รูปควรจัดวางตัวสินค้าให้อยู่ในไฟล์อย่างเหมาะสม มีขอบซ้าย/ขวา/บน/ล่างพอดี
- หน้าเว็บต้องแสดงรูปตามไฟล์จริง
- ระบบไม่ควร crop รูปสินค้าอัตโนมัติจนตัวสินค้าโดนตัด
- การแสดงผลควรเป็นแนวทางที่เห็นสินค้าเต็มภาพ

### สิ่งที่หลังบ้านต้องบอกแอดมิน

- ขนาดรูปแนะนำ
- สัดส่วนรูปแนะนำ
- รูปนี้ใช้ทั้ง desktop และ mobile
- ถ้าต้องการให้สินค้าใหญ่ขึ้นหรือเล็กลง ต้องจัดตำแหน่งในไฟล์ต้นทางเอง
- ไม่ควรอัปโหลดรูปที่ตัวสินค้าชิดขอบเกินไป
- ถ้ารูปผิดสัดส่วน เว็บอาจแสดงผลไม่ตรงตามที่ต้องการ

### จุดที่ต้องใช้กฎนี้ร่วมกัน

- หน้าเพิ่มสินค้า
- หน้าแก้ไขสินค้า
- หน้าแสดงสินค้าในหลังบ้าน
- หน้าแสดงสินค้าเว็บหน้าบ้าน
- รูป detail/gallery ของสินค้า

## 5. Product / ข้อมูลสินค้าที่แสดงบนเว็บ

ถึงแม้หน้านี้จะเป็น Website Editor แต่ระบบต้องเชื่อมโยงกับสินค้า เพราะสินค้าเป็นข้อมูลหลักที่แสดงบนเว็บ

ข้อมูลสินค้าที่ระบบหลังบ้านต้องจัดการได้:

- ชื่อสินค้า
- ราคา
- สี
- ไซซ์
- หมวดหมู่
- รูปหลัก
- รูป detail/gallery เพิ่มได้หลายรูป
- สถานะแสดงบนเว็บ
- slug หรือ URL ของสินค้า
- รายละเอียดสินค้า
- variant เช่น สี + ไซซ์
- จำนวนสต็อกของแต่ละ variant

## 7. การเชื่อมสินค้าเข้ากับสต็อก

ส่วนนี้สำคัญมาก เพราะไม่ควรให้แอดมินเพิ่มสินค้าในหน้าสินค้า แล้วต้องไปเพิ่มซ้ำในหน้าสต็อกอีกครั้ง

### หลักการ

เมื่อแอดมินสร้างสินค้า 1 ตัว ระบบควรสร้างข้อมูล stock ให้พร้อมโดยอัตโนมัติจากสีและไซซ์ที่ใส่ไว้

ตัวอย่าง:

สินค้า: Retro Patchwork Striped Color Block Short

สี:

- Red
- Black

ไซซ์:

- S
- M
- L
- XL

ระบบต้องสร้าง variant/stock เช่น:

- Red / S
- Red / M
- Red / L
- Red / XL
- Black / S
- Black / M
- Black / L
- Black / XL

แต่ละ variant ต้องมีจำนวนสต็อกของตัวเอง

### สิ่งที่ต้องแก้ไขได้

- เพิ่มสี
- ลบสี
- แก้ชื่อสี
- กำหนด swatch สี
- เพิ่มไซซ์
- ลบไซซ์
- แก้ชื่อไซซ์
- ใส่จำนวนสต็อกต่อสี/ไซซ์
- ตั้ง minimum stock
- เปิด/ปิด variant
- กำหนดว่าตัวไหนหมด

### พฤติกรรมเมื่อสีหรือไซซ์หมด

ถ้าสีหรือไซซ์ใดหมด:

- หน้าเว็บยังควรแสดงสินค้านั้นอยู่
- ตัวเลือกสี/ไซซ์ที่หมดต้องขึ้นว่า “หมดแล้ว”
- ผู้ใช้ต้องกดเลือกซื้อไม่ได้
- ปุ่ม add to cart หรือ checkout ต้องไม่ทำงานกับ variant ที่หมด
- หลังบ้านต้องเห็นชัดว่า variant ไหนหมด

### พฤติกรรมเมื่อสินค้าหมดทุก variant

ถ้าสินค้าทุกสี/ทุกไซซ์หมด:

- สินค้ายังแสดงบนเว็บได้
- ต้องมีสถานะ “หมดแล้ว”
- กดซื้อไม่ได้
- แอดมินสามารถเลือกซ่อนสินค้าเองได้ ถ้าไม่ต้องการให้แสดง

## 8. รูปสินค้าและ Gallery

สินค้า 1 ตัวต้องมีรูปได้หลายรูป

### รูปที่ต้องรองรับ

- รูปหลักบนการ์ดสินค้า
- รูปหน้ารายละเอียดสินค้า
- รูป gallery เพิ่มเติม
- รูปแยกตามสี ถ้าจำเป็น

### สิ่งที่ต้องทำได้

- เพิ่มรูป
- ลบรูป
- จัดลำดับรูป
- ตั้งรูปหลัก
- ใส่ alt text
- preview รูป
- แสดงคำแนะนำขนาดรูป

### กฎการใช้รูป

- ใช้ขนาดแนะนำ 1920 × 2400 px
- ใช้ไฟล์เดียวสำหรับ desktop/mobile
- เว็บหน้าบ้านต้องแสดงสินค้าเต็มตามที่จัดวางไว้ในไฟล์
- ห้าม crop จนตัวสินค้าหาย

---

## 9. Preview ก่อนบันทึก

ระบบควรมี preview เพื่อให้แอดมินเห็นก่อนบันทึก

Preview ที่ควรมี:

- preview cover image/video
- preview intro image/video
- preview รูปสินค้า
- preview สถานะเปิด/ปิด
- preview ข้อมูลที่จะส่งผลกับหน้าเว็บ

Preview ไม่จำเป็นต้องเหมือนหน้าเว็บจริง 100% แต่ต้องช่วยให้แอดมินตรวจสอบข้อมูลก่อนกดบันทึกได้

---

## 10. การบันทึกและสถานะข้อมูล

ทุกส่วนที่แก้ไขต้องมีการบันทึกชัดเจน

ระบบควรรองรับ:

- บันทึก cover media
- บันทึก intro media
- บันทึกกฎรูปสินค้า
- บันทึกข้อมูลสินค้า/variant/stock ผ่านระบบสินค้า
- แจ้งเตือนเมื่อบันทึกสำเร็จ
- แจ้งเตือนเมื่อบันทึกไม่สำเร็จ
- ป้องกันการออกจากหน้าถ้ามีข้อมูลที่แก้แล้วยังไม่บันทึก

สถานะที่ควรมี:

- Active
- Inactive
- Hidden
- Sold out

---

## 11. สิทธิ์การใช้งาน

หน้านี้ควรใช้ได้เฉพาะผู้ใช้หลังบ้านที่มีสิทธิ์

สิทธิ์ที่ควรแยก:

- ดูข้อมูลได้
- จัดการ cover media ได้
- จัดการ intro media ได้
- แก้ไขสินค้าได้
- แก้ไขสต็อกได้

ถ้ามี role หลายระดับ เช่น owner, admin, staff ควรกำหนดสิทธิ์ให้ชัดเจน

## 12. ข้อมูลที่ต้องเชื่อมกับฐานข้อมูล

ข้อมูลของหน้านี้ไม่ควรเป็น mock data ถาวร

ข้อมูลที่ต้องเก็บจริง:

- cover media
- cover media type
- cover active status
- intro media
- intro media type
- intro active status
- product image rule
- product categories
- product media
- product variants
- stock quantity
- stock movement
- sold out status

---

## 13. ขอบเขตที่ยังไม่ต้องทำตอนนี้

เพื่อให้ scope ไม่ใหญ่เกินไป รอบแรกยังไม่ต้องรวม:

- แก้ข้อความบน cover
- แก้ปุ่ม CTA บน cover
- แก้ link บน cover
- ตั้งค่าเสียง intro
- ตั้งค่า fade intro
- ตั้งค่าระยะเวลา intro
- ตั้งค่า intro แสดงครั้งเดียว
- ระบบ crop รูปขั้นสูง
- ระบบ schedule campaign
- ระบบ A/B testing
- ระบบแปลหลายภาษา
- ระบบ approval ก่อน publish
- ระบบ version history
- ระบบ rollback

## 14. สรุป

หน้า Website Editor ต้องเป็นหน้าสำหรับจัดการส่วนที่แสดงบนเว็บ SHOW OFF โดยเน้น:

- เพิ่ม/แก้ไข/ลบ cover รูปภาพและวิดีโอ
- เพิ่ม/แก้ไข/ลบ intro รูปภาพและวิดีโอ
- กฎรูปสินค้า
- หมวดหมู่สินค้า
- การเชื่อมสินค้าเข้ากับสต็อก
- การแสดงสถานะสินค้าหมดบนเว็บ

สิ่งสำคัญที่สุดคือ ระบบนี้ต้องลดการทำงานซ้ำของแอดมิน

เมื่อเพิ่มสินค้า 1 ครั้ง ข้อมูลควรเชื่อมไปที่:

- หน้าเว็บ
- หมวดหมู่สินค้า
